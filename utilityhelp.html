Help for /utility route

Last updated by Akash Shanmugaraj on 30th May 2023

This route is used for all means of cross verification of data for frontend users.

The current version is meant to be a non deployable testing server model, and will not used in the production server.

Available routes:

- GET {BASE_URL}/utility/ sample root API

- GET {BASE_URL}/utility/room/all returns all room data currently available

- GET {BASE_URL}/utility/timetable/all returns all timetable data currently available

- GET {BASE_URL}/utility/course/all returns all course data currently available

- GET {BASE_URL}/utility/attendance/all returns all attendance data currently available

- GET {BASE_URL}/utility/user/all returns all room user currently available

- POST {BASE_URL}/utility/info returns how the server recieves your request, better suited for debugging

- GET {BASE_URL}/utility/log/info returns all logs of the level info

Currently the application is only configured to stream info-level logs.
