Help for /auth route

Last updated by Akash Shanmugaraj on 24th May 2023

This route is used for all means of authentication purposes. The current version is meant to be a non deployable testing server model, and will not be exactly used in the production server.

Available routes:

- GET {BASE_URL}/authentication/ sample root API

- POST {BASE_URL}/authentication/verify is used for verifying credentials of users, provide rollnumber, tutorcode and password as body parameters in your HTTP request. 
  This route returns a boolean response

Testing

Routes marked as PROTECTED can be used for testing the authentication perspective of the application

Use 22Z201 as rollnumber, 1234 as password

Previously available authentication features such as tokens and encrypted-authentication will be rolled out soon on a need
