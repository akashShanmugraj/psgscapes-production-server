Help for /auth route

This route is used for all means of authentication purposes. The current version is meant to be a non deployable testing server model, and will not be exactly used in the production server.

Available routes:

- GET {BASE_URL}/auth/ (PROTECTED) is a sample root API 

- POST {BASE_URL}/auth/login is used for login of new users, provide rollnumber, tutorcode and password as headers in your HTTP request. 
  This route returns token and authenticationtext which must be stored.

- POST {BASE_URL}/auth/renew is used for renewing expired tokens, provide rollnumber, token (the expired token), and auth (authentication text returned from latest renewal or login).
  This route returns a new set of token and authenticationtext

- GET {BASE_URL}/info (ONLY TESTING) is used to help a developer know how their HTTP request(s) are recieved.
  Displays several metrics of the request. No specific API configuration required
